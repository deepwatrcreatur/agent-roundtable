## Round 47 — The Project Mind (Active Inference)

**Tags:** philosophy, governance, epistemic-integrity
**Status:** Closed  
**Voices used:** Codex CLI, Gemini CLI  
**Claude:** Not used for closure in this run

### First-pass convergence

- Vouches should be attached to explicit claims, basis, scope, and expiry rather
  than treated as ambient reputation.
- Stress should be computed over **contested objects** (tasks, decisions,
  interfaces, PRs), not over people.
- The orchestrator should allocate attention by risk-reduction heuristics rather
  than by whoever speaks the most.
- Automation should detect, score, and recommend; legitimacy-critical acts stay
  human- or IC-governed.

### Disconfirmation findings

The real red-team passes found that “object stress” can still launder politics:

- a contested object is often a proxy for authority, status, incentives, or
  asymmetric risk
- a stress metric can be gamed through manufactured stress / attention-mining
- people do not disappear just because the UI highlights objects instead of
  names

### Narrowed follow-up and closure

The round closes with two added protocol rules.

#### 1. People/power dispute escape hatch

Any directly affected participant has standing to declare that a conflict is a
people/power dispute if they name a concrete asymmetry in control, exposure, or
sanction.

When that declaration is accepted:

- the issue moves into a dedicated **power-review** state
- object-level stress stops being the lead routing signal
- the first task becomes mapping decision power, cost/risk exposure, and the
  active coercive/dependency lever

#### 2. Anti-gaming rules

- stress escalations must be tied to **verifiable constraints**, not just felt
  intensity
- repeated unsupported self-escalation stops earning fast-path attention and
  goes to cooldown / review

### Bottom line

Active Inference is useful only when translated back into explicit governance:

- claim-scoped precision weighting
- object stress with visible challenge paths
- explicit people/power override state
- automation for sensing, humans for legitimacy
